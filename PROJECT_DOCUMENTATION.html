Operational Mechanism

01

Identity & Persistence

The application utilizes a persistent identity layer. On login, users register their name and phone. All conversations are stored in a centralized SQLite database, allowing the farmer to switch between 'Dashboard' and 'Chat' without losing their current dialogue or historical insights.

02

Linguistic Continuity & Script-Locking

An advanced script-matching engine scans for Latin vs. Devanagari characters. If a user types in Hinglish using English characters, the AI is "clamped" into Latin-only output, ensuring the advisor matches the energy and script style of the rural user.

03

Visual Optimization Pipeline

Strategy reports are processed through an aesthetic filter that strips raw markdown symbols (**), reduces background noise opacity (reduced to 0.04), and presents dense advice as clean, digestible cards for mobile-first accessibility.

04

Uninterruptible Intelligence

Our autonomous failover logic seamlessly manages a pool of high-capacity Gemini 2.5 Flash keys. In the rare event of a node timeout, the system recovers in milliseconds, ensuring the advisor is always ready to help.

Developer Analysis & Demo Guide

Architecture: Lifted State Persistence

A key highlight for developers is the Lifted State Management. By moving the chat and loading states to App.tsx, we ensure that a user can browse their mandi report and switch back to the AI chat without re-rendering or re-fetching. This mimics a native app experience where memory is preserved across views.

The Script-Match Engine

Explain the is_latin scanning logic in the backend. By identifying the character set (Unicode Range) of the input, we dynamically append script-locking instructions to the system prompt, preventing the common AI flaw of switching to Devanagari script mid-conversation.

Visual Polish: "Wow" Effect Mechanics

Mention the custom renderMessage logic which uses regex to transform raw markdown bullets and headers into high-fidelity UI elements. This ensures that even if the AI outputs standard text, it appears on the frontend as a designer-crafted layout.
